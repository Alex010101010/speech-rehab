# Правила проекта speech-rehab

## Деплой: OTA vs APK — предупреждать в чате

**Перед деплоем явно предупреждать в чате, если правки требуют НЕ ТОЛЬКО OTA-обновления.**

- **OTA** (GitHub Pages `/ota`, без пересборки APK) доставляет на планшет **только контент**: `content/json/*.json`, `content/img/*`, `content/index.json` (при выпуске поднимать `content_version`).
- **НЕ доставляется по OTA** (нужен новый APK): любой код в `app/lib/**` (новые/изменённые виджеты, движок, экраны), `kAppOtaVersion`/`min_app_version`, зависимости `pubspec`.
- Поэтому: если в изменениях затронут `app/lib/**` → перед деплоем сказать пользователю «эти правки — код, OTA их не доставит, нужна пересборка APK (`gh workflow run build.yml -f build_apk=true`) и установка на планшет», и только потом деплоить.

## Пуш и сборка — только по явному слову

- `git push` и сборку APK делать **только** по явному слову пользователя («пушим», «деплой», «собираем APK»). Не автоматически.
- Локального Flutter НЕТ → компиляцию Dart проверяет ТОЛЬКО CI (web-сборка на push). На push собирается только web; APK — вручную.
