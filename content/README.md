# Контент-пак тренажёра — профиль и схема

> ⚠️ ЧЕРНОВИК. Весь сгенерированный контент **обязателен к проверке логопедом** перед использованием.

## Профиль пациента (для тематической привязки)
- Мужчина 70+, **лёгкая форма афазии** после инсульта, прогноз реабилитации удовлетворительный.
- Профессия — **сварщик**. Хобби — **рыбалка**.
- Любил **читать книги** (особенно **Стивена Кинга**).
- Последние годы до инсульта в основном дома: **бытовые занятия + телевизор**; смотрит **ТВ / аниме / корейское / китайское**.
- Рядом жена 70+ как соведущий.

## Вес тем
- **Ядро** словарных и грамматических заданий: **сварка + рыбалка + быт + природа** (преморбидная лексика восстанавливается лучше всего — принцип Оппеля).
- **ТВ / аниме / дорамы / «кинговские» сюжеты** — только для **текстов на чтение и пересказ** и связной речи, НЕ для словарных дрилл (имена/иностранное не годятся в дрилл).
- Тег `theme`: `сварка | рыбалка | быт | природа | тв_кино | универсальное`.
- Уровень `level`: `1` (проще) … `3` (сложнее) в пределах лёгкой формы.

## Схема данных (JSON)
Каждый тип — отдельный файл `content/json/NN_<type>.json`:
```json
{ "type": "<machine_id>", "title": "<рус. название>", "section": "<I..VII>", "count": 0, "items": [ /* ... */ ] }
```
Для заданий со свободным вводом ответа — массив `accept` с допустимыми вариантами (мягкая автопроверка, принцип Оппеля «лучше подсказать, чем мучительный поиск»).

## На вычитку логопеду (отмечено агентами)
- Несколько вольных формулировок пословиц (05) — проверить узнаваемость.
- Широкие списки `accept` в словарных типах (06–10) и спорные ударения (16: творо́г/тво́рог, ёрш) — подтвердить допустимость.
- `level` в грамматике/звуках (14–17) распределён неравномерно (формат тяготеет к 1–2) — при желании перебалансировать.

## Ре-разметка level по фидбеку методиста (черновик, на валидацию)
Применены прозрачные правила, **только повышение** сложности (ничего не понижалось):
- **R1 — память (19):** ряд ≥5 элементов → level ≥3; ряд =4 → level ≥2 (старт 3–4, 5+ — поздний этап). Поднято 14 заданий.
- **R2 — незаконченные фразы (02):** многопридаточные/длинные → level ≥3 (запятая + подчинит. союз «котор…/потому что/поэтому/чтобы/хотя/несмотря/так как/благодаря», либо ≥9 слов). Поднято 16 заданий.
- **R3 — advanced-типы:** `proverbs` (05), `paronyms` (12), `word_formation` (11), `stress` (16) помечены `advanced:true` в `index.json` — абстракция/переносный смысл, вне базового потока (в ротацию сессии не входят; для лёгкой афазии по выбору логопеда).
- **R4 — чтение (18):** level назначен по длине текста (число слов), отсортировано и разбито на 3 равные группы → короткие тексты первыми. Здесь переразметка полная (длина — главный драйвер сложности чтения+пересказа).
- **R5 — короткие тексты (18, rt_021..026):** добавлено 6 текстов на нижний уровень (3 предложения, 6–12 слов, 2 вопроса, ответы 1–3 слова), level 1 — черновик, на валидацию. Прежний минимум был 6–8 предложений / 31+ слово.
- **R6 — обобщения (08):** 7 явно абстрактных/узкоспециальных категорий («отходы сварки», «сварочное снаряжение», «печатная продукция», «почтовые отправления», «культурные учреждения», «обитатели пруда/реки») подняты на L3 (promote-only, judgment-call — на валидацию; полная семантическая разметка в BACKLOG).
- **memory_rows (19):** к рядам с цифрами добавлено поле `display` (например `"5 (пять)"`) — приложение показывает цифру + пропись; озвучка читает число.
- **Поведение приложения (не контент):** на L1 чтение показывает 2 вопроса, L2 — 3, L3 — все; пересказ (связная речь) — только на L3.
- **Ввод по клеткам (показ формы):** в текстовых заданиях над полем рисуются пустые клетки = число букв в каждом слове эталона `answer` (опора при афазии). Ввод по-прежнему в одно поле с мягкой проверкой. Не показывается для `fill_letter` (своя форма), errorless-шагов и длинных ответов (>6 слов или слово >14 букв).
- **Чтение (18) — короткие ответы на L1:** ответы вопросов у 6 старых L1-текстов сокращены до коротких фраз (1–3 слова); новые rt_021..026 уже короткие. L2/L3 — без изменений (на потом).
- **Новый тип `clock` (21):** «Сколько времени» — показ электронных часов (HH:MM), устный ответ с озвучкой; 14 заданий (L1 ровные часы → L3 произвольные минуты). Черновик, на валидацию.
- **Новый тип `picture_word` (22) — этаж L0 «картинка→слово», `draft:true`:** до-вербальный пол лестницы (узнавание, не воспроизведение) для словарных навыков (name_by_description, fill_letter, generalization, synonyms_antonyms). 63 задания: картинка + 3 слова на выбор. Сгенерирован детерминированно из одобренных пар картинка→слово в `fill_letter` (`tools/build_picture_word.py`), без LLM-выдумок. Дистракторы — другие слова пула без совпадения первой буквы/рифмы (опора на смысл). Картинки добавляются позже через `EXTRA_PAIRS` в скрипте или новые `image` в fill_letter → перезапуск генератора. **На валидацию логопедом:** соответствие картинка↔слово и подбор дистракторов.

**Требует ручной валидации логопедом (правилами не размечалось):**
- Абстрактные пары синонимов/антонимов (10), помеченные L1–L2, — возможна недооценка сложности.
- Поэлементная абстрактность пословиц/паронимов внутри advanced-типов.
